Login/Register

Action

POST /auth/login

Request

  • email - User's email address.
  • username - User's username.
  • device_token - device token (push notification)

Response

{
    "id": "1",
    "email": "saakyz@gmail.com",
    "username": "n3tr",
    "token": "21345o34567899765thsdfgdsasrgfdcvtfbcv4",
    "created_at": "2013-08-23 07:22:36",
    "updated_at": "2013-08-23 07:22:36"
}

Events

Action

GET /event/create

Request

  • access_token - !important: User's email
  • where_text - String to define context of event.
  • location - location string.

Response

{
    "where_text": "test",
    "start_point": 5,
    "status": 1,
    "user_id": 3,
    "location": "",
    "updated_at": "2013-08-23 17:19:44",
    "created_at": "2013-08-23 17:19:44",
    "id": 1
}

Favour Create

Create Favour

** Action **

GET /favour/create

** Request Params **

  • event_id: event's id which need to create apply favour to.
  • access_token: user's email.

** Response **

{
    "event_id": "1",
    "from_user_id": "1",
    "status": 0,
    "bid_point": "5",
    "updated_at": "2013-08-23 20:25:47",
    "created_at": "2013-08-23 20:25:47",
    "id": 2,
    "user_event": {
        "id": "1",
        "user_id": "3",
        "where_text": "Wher aaf0aojfoa",

        "status": "1",

Accept Favour

Accept Favour (Change Favour status to 1)

** Action **

GET /favour/create

** Request Params **

  • favour_id: id of favour which want to accept.
  • access_token: user's email.

** Response **

{
    "event_id": "1",
    "from_user_id": "1",
    "status": 1,
    "bid_point": "5",
    "updated_at": "2013-08-23 20:25:47",
    "created_at": "2013-08-23 20:25:47",
    "id": 2,
}

Reject Favour

Reject Favour (Change Favour state to 2)

** Action **

** Request Params **
